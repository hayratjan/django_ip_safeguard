# 02 安装与环境要求

## 2.1 运行环境

| 组件 | 要求 | 说明 |
|------|------|------|
| Python | `>= 3.9` | 与 `pyproject.toml` 中 `requires-python` 一致 |
| Django | `>= 4.2` | LTS 建议 4.2.x |
| Redis | 建议 6.x+ | 情报缓存、封禁、限流、地理池索引、熔断计数均依赖 Redis |
| 可选 | Node.js 18+ | 仅构建/开发 `frontend-admin` 时需要 |

---

## 2.2 安装 Python 包

```bash
pip install django-ip-safeguard
```

或从源码目录安装（可编辑模式便于开发）：

```bash
pip install -e ".[dev]"
```

**主要依赖**（随包安装）：`Django`、`redis`、`httpx`、`PyJWT`、`django-unfold`。  
**可选依赖**：
- `[geoip2]`：安装 `geoip2` 包以支持 MaxMind GeoLite2 本地离线查询，获取精确的国家/城市/ASN 信息。
  ```bash
  pip install django-ip-safeguard[geoip2]
  ```
开发依赖（`[dev]`）：`pytest`、`pytest-django`、`ruff`、`build`、`twine` 等。

---

## 2.3 注册 Django 应用

在项目的 `settings.py` 中：

```python
INSTALLED_APPS = [
    # ...
    "unfold",
    "unfold.contrib.filters",
    "django_ip_safeguard",
]
```

执行数据库迁移（审计、封禁、策略、地理池状态等表）：

```bash
python manage.py migrate django_ip_safeguard
```

---

## 2.4 注册中间件

在 `settings.py` 的 `MIDDLEWARE` 中，将 **`IpGuardMiddleware`** 放在尽可能靠前、但又在 `SecurityMiddleware` / `CommonMiddleware` 等之后的合理位置。典型做法：

```python
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "django.contrib.sessions.middleware.SessionMiddleware",
    "django.middleware.common.CommonMiddleware",
    "django_ip_safeguard.middleware.IpGuardMiddleware",  # 建议靠前后续可拿到 Session 等
    # "django.middleware.csrf.CsrfViewMiddleware",
    # ...
]
```

**说明**：

- 若需在中间件之后仍能使用 Django 认证结果，请勿放在 `AuthenticationMiddleware` 之前（当前插件中间件**不依赖**登录用户，仅依赖 IP 与配置）。  
- 若全站 API 均为 JSON，拦截响应为 `JsonResponse`，与前后端分离架构兼容。

## 2.5 挂载 URL

在项目根 `urls.py`：

```python
from django.urls import include, path

urlpatterns = [
    path("ip-guard/", include("django_ip_safeguard.urls")),
]
```

则所有插件路由前缀为 **`/ip-guard/`**（若需修改前缀名，见配置项 `IP_GUARD_ADMIN_URL_PREFIX` 与文档 [04](./04-配置项完整参考.md)）。

## 2.7 常见问题

**Q：未装 Redis 能否启动？**  
A：中间件会访问 Redis；无 Redis 时相关操作会失败并可能触发失败放行逻辑，**不建议**在无 Redis 环境启用插件。

**Q：与 `django-ratelimit` 冲突吗？**  
A：不冲突；本插件的限流在 IP 维度、且发生在业务视图之前，与视图层限流可并存，注意误双重拦截的策略设计。

**Q：静态资源是否会被拦？**  
A：中间件默认处理所有进入 Django 的请求；若静态文件经 Whitenoise 或 CDN 同域走 Django，请用白名单或路径规则放行。推荐静态资源独立域名或 CDN，不经本中间件。

## 2.8 多语言（i18n）配置

Django IP Safeguard（IP安全卫士）已内置国际化支持，默认语言为简体中文。

### 启用多语言

在 `settings.py` 中确保以下配置：

```python
LANGUAGE_CODE = 'zh-hans'

LANGUAGES = [
    ('zh-hans', '简体中文'),
    ('en', 'English'),
]

USE_I18N = True

LOCALE_PATHS = [
    BASE_DIR / 'locale',
]
```

### 翻译文件位置

- 插件自带翻译：`django_ip_safeguard/locale/zh_Hans/LC_MESSAGES/django.po`
- 项目级翻译（覆盖 unfold 等第三方模板字符串）：`demo_project/locale/zh_Hans/LC_MESSAGES/django.po`

### 编译翻译文件

修改 `.po` 文件后需重新编译：

```bash
python manage.py compilemessages
# 或手动
msgfmt locale/zh_Hans/LC_MESSAGES/django.po -o locale/zh_Hans/LC_MESSAGES/django.mo
```

### 添加新语言

1. 创建目录：`mkdir -p django_ip_safeguard/locale/<lang_code>/LC_MESSAGES`
2. 复制并编辑 `.po` 文件
3. 编译：`msgfmt django.po -o django.mo`
4. 在 `LANGUAGES` 中添加新语言选项
