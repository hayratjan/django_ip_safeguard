# 16 - 多语言（i18n）配置与切换

## 概述

Django IP Safeguard 支持中文（简体）与英文两种语言的实时切换，覆盖 **Django Admin 后台** 和 **Vue 前端管理控制台** 两个界面。语言切换后前后端同步生效，无需刷新页面。

---

## 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                     用户点击语言切换                        │
│                 (AppLayout / LoginView)                  │
└───────────────────────┬─────────────────────────────────┘
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
   vue-i18n 切换   i18n Store    POST /i18n/switch/
   前端文本即时更新  localStorage   Django session + cookie
                    持久化选择       后端语言同步切换
          │                            │
          ▼                            ▼
   Accept-Language 请求头        Django LocaleMiddleware
   (HTTP 拦截器自动附加)          自动识别并激活语言
```

## 一、Django 后端国际化

### 1.1 Settings 配置

在 `demo_project/demo_site/settings.py` 中：

```python
# 国际化开关
USE_I18N = True

# 默认语言
LANGUAGE_CODE = 'zh-hans'

# 支持的语言列表
LANGUAGES = [
    ('zh-hans', '简体中文'),
    ('en', 'English'),
]

# 翻译文件搜索路径
LOCALE_PATHS = [
    BASE_DIR / 'locale',
    BASE_DIR.parent / 'django_ip_safeguard' / 'locale',
]

# 中间件顺序（LocaleMiddleware 必须在 SessionMiddleware 之后、CommonMiddleware 之前）
MIDDLEWARE = [
    ...
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.locale.LocaleMiddleware',
    ...
]
```

### 1.2 翻译文件结构

```
django_ip_safeguard/
└── locale/
    ├── en/LC_MESSAGES/
    │   ├── django.po    # 英文翻译源文件
    │   └── django.mo    # 编译后的二进制文件
    └── zh_Hans/LC_MESSAGES/
        ├── django.po    # 中文翻译源文件
        └── django.mo    # 编译后的二进制文件

demo_project/
└── locale/
    ├── en/LC_MESSAGES/
    │   ├── django.po
    │   └── django.mo
    └── zh_Hans/LC_MESSAGES/
        ├── django.po
        └── django.mo
```

### 1.3 翻译标记

在 Python 代码中使用 `gettext_lazy`（别名 `_`）标记需要翻译的字符串：

```python
from django.utils.translation import gettext_lazy as _

def api_error(message: str, ...):
    ...

# 使用示例
return api_error(_("未登录或登录已过期"), code=4010, status=401)
return api_error(_("权限不足"), code=4030, status=403)
```

### 1.4 语言切换 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/ip-guard/api/i18n/languages/` | GET | 获取支持的语言列表及当前语言 |
| `/ip-guard/api/i18n/switch/` | POST | 切换当前会话语言 |

**GET /i18n/languages/ 响应示例：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "languages": [
      {"code": "zh-hans", "name": "简体中文"},
      {"code": "en", "name": "English"}
    ],
    "current": "zh-hans"
  }
}
```

**POST /i18n/switch/ 请求体：**

```json
{
  "language": "en"
}
```

切换成功后会：
1. 调用 `activate()` 激活 Django 翻译
2. 写入 Session（`django_language` key）
3. 设置 Cookie（`django_language`，默认有效期 1 年）

### 1.5 翻译文件编译

修改 `.po` 文件后需要重新编译为 `.mo` 文件：

```bash
# 方式一：使用 Django 命令
python manage.py compilemessages

# 方式二：使用 msgfmt 直接编译
msgfmt django_ip_safeguard/locale/en/LC_MESSAGES/django.po \
     -o django_ip_safeguard/locale/en/LC_MESSAGES/django.mo
msgfmt django_ip_safeguard/locale/zh_Hans/LC_MESSAGES/django.po \
     -o django_ip_safeguard/locale/zh_Hans/LC_MESSAGES/django.mo
```

### 1.6 提取新翻译字符串

当代码中新增了 `_()` 标记的字符串后：

```bash
# 从 Python 代码中提取待翻译字符串
python manage.py makemessages -l en -l zh_Hans

# 编译
python manage.py compilemessages
```

## 二、Vue 前端国际化

### 2.1 技术栈

- **vue-i18n** (^9.x)：Vue 3 官方国际化插件
- **Pinia Store**：管理语言状态，同步前后端
- **Element Plus Locale**：UI 组件库国际化

### 2.2 文件结构

```
frontend-admin/src/
├── i18n/
│   ├── index.js     # i18n 实例配置、语言检测逻辑
│   ├── zh.js        # 中文翻译字典
│   └── en.js        # 英文翻译字典
├── stores/
│   └── i18n.js      # Pinia Store - 语言状态管理
├── api/
│   ├── index.js     # i18n API 调用封装
│   └── http.js      # Axios 拦截器（含 Accept-Language）
├── App.vue          # ElConfigProvider 动态 locale
└── main.js          # i18n 插件注册
```

### 2.3 i18n 实例配置

`src/i18n/index.js`：

```javascript
import { createI18n } from "vue-i18n";
import zh from "./zh";
import en from "./en";

const STORAGE_KEY = "ip_guard_locale";

function detectLocale() {
  // 1. 优先读取 localStorage
  const stored = localStorage.getItem(STORAGE_KEY);
  if (stored && (stored === "zh" || stored === "en")) return stored;
  // 2. 根据浏览器语言自动检测
  const navLang = navigator.language || navigator.userLanguage || "";
  if (navLang.toLowerCase().startsWith("zh")) return "zh";
  return "en";
}

const i18n = createI18n({
  legacy: false,           // 使用 Composition API 模式
  locale: detectLocale(),  // 初始语言
  fallbackLocale: "zh",    // 回退语言
  messages: { zh, en },
});
```

### 2.5 Pinia i18n Store

`src/stores/i18n.js`：

```javascript
export const useI18nStore = defineStore("i18n", {
  state: () => ({
    currentLocale: i18n.global.locale.value,
    languages: [
      { code: "zh", name: "简体中文" },
      { code: "en", name: "English" },
    ],
  }),
  actions: {
    // 仅切换前端语言
    setLocale(locale) {
      this.currentLocale = locale;
      i18n.global.locale.value = locale;
      localStorage.setItem(STORAGE_KEY, locale);
      document.documentElement.setAttribute("lang", locale === "zh" ? "zh-Hans" : "en");
    },
    // 前后端同步切换
    async switchLocale(locale) {
      this.setLocale(locale);
      try {
        const backendLang = locale === "zh" ? "zh-hans" : "en";
        await i18nLangSwitchApi({ language: backendLang });
      } catch { /* 静默忽略后端切换失败 */ }
    },
  },
});
```

### 2.6 组件中使用翻译

在 Vue 组件中使用 `useI18n` 的 `t` 函数：

```vue
<template>
  <h3>{{ t('dashboard.title') }}</h3>
  <el-button @click="load">{{ t('common.refresh') }}</el-button>
</template>

<script setup>
import { useI18n } from "vue-i18n";
const { t } = useI18n();
</script>
```

带参数的翻译：

```vue
<span>{{ t('health.latency', { ms: data.redis_latency_ms }) }}</span>
```

### 2.7 Element Plus 国际化

通过 `ElConfigProvider` 在 `App.vue` 中实现动态切换：

```vue
<template>
  <el-config-provider :locale="elementLocale">
    <router-view />
  </el-config-provider>
</template>

<script setup>
import { computed } from "vue";
import zhCn from "element-plus/es/locale/lang/zh-cn";
import en from "element-plus/es/locale/lang/en";
import { useI18nStore } from "./stores/i18n";

const i18nStore = useI18nStore();
const elementLocale = computed(() =>
  i18nStore.currentLocale === "zh" ? zhCn : en
);
</script>
```

这确保 Element Plus 的分页、日期选择器、表格空数据等内置文本也跟随语言切换。

### 2.8 HTTP 请求自动附加语言头

`src/api/http.js` 中的 Axios 请求拦截器自动添加 `Accept-Language` 头：

```javascript
http.interceptors.request.use((config) => {
  // ... JWT token 处理 ...

  const savedLang = localStorage.getItem("ip_guard_locale");
  if (savedLang) {
    config.headers["Accept-Language"] = savedLang === "zh" ? "zh-hans" : "en";
  }
  return config;
});
```

Django 的 `LocaleMiddleware` 会读取该请求头，自动激活对应语言的翻译。

### 2.9 语言切换 UI

**已登录页面（AppLayout）**：顶部右侧 Header 区域，地球图标下拉菜单。

**登录页面（LoginView）**：表单上方右侧，地球图标下拉菜单。

切换流程：
1. 用户点击语言选项
2. `i18nStore.switchLocale(locale)` 被调用
3. 前端 vue-i18n 立即切换，界面文本实时更新
4. Element Plus locale 通过 computed 响应式更新
5. 后端 API 被调用，Session 和 Cookie 同步更新
6. 后续 HTTP 请求自动携带新的 `Accept-Language` 头

## 三、Django Admin 后台国际化

Django Admin 自带完整的国际化支持。当 `USE_I18N = True` 且 `LocaleMiddleware` 启用后：

1. Admin 界面会根据请求的语言自动显示对应翻译
2. 通过前端调用 `/i18n/switch/` API 切换语言后，Admin 页面也会同步切换
3. 也可以直接在 Admin URL 中使用 `/en/admin/` 或 `/zh-hans/admin/` 前缀切换

## 四、添加新语言

如需支持更多语言（如日语），步骤如下：

### 4.1 Django 后端

1. 在 `settings.py` 的 `LANGUAGES` 列表中添加新语言：
   ```python
   LANGUAGES = [
       ('zh-hans', '简体中文'),
       ('en', 'English'),
       ('ja', '日本語'),
   ]
   ```

2. 生成翻译文件：
   ```bash
   python manage.py makemessages -l ja
   ```

3. 编辑 `locale/ja/LC_MESSAGES/django.po`，填入日文翻译

4. 编译：
   ```bash
   python manage.py compilemessages
   ```

### 4.2 Vue 前端

1. 创建 `src/i18n/ja.js` 翻译字典
2. 在 `src/i18n/index.js` 中导入并注册
3. 在 `src/stores/i18n.js` 的 `languages` 数组中添加新选项
4. 在 `App.vue` 中导入 Element Plus 的日文 locale

## 五、注意事项

1. **翻译文件编译**：修改 `.po` 文件后必须重新编译 `.mo` 文件，否则翻译不会生效
2. **缓存问题**：Django 可能缓存翻译，开发时设置 `DEBUG = True` 可避免
3. **前端语言持久化**：语言选择存储在 `localStorage` 的 `ip_guard_locale` key 中
4. **后端语言持久化**：通过 Django Session 和 Cookie 双重机制持久化
5. **语言代码映射**：前端使用 `zh` / `en`，Django 使用 `zh-hans` / `en`，Store 中自动转换
6. **Element Plus locale**：必须通过 `ElConfigProvider` 的 `:locale` prop 动态绑定，不能在 `main.js` 中静态配置
