# 14 管理命令与定时任务

## 14.1 Django 迁移

安装或升级插件版本后：

```bash
python manage.py migrate django_ip_safeguard
```

会创建/更新包括但不限于：`IpGuardPolicy`、`IpAccessLog`、`IpBanRecord`、`IpGeoPoolStatus`、`IpReputationHistory`、`ThreatIntelFeedStatus` 等表。

## 14.2 地理 IP 池同步：`sync_geo_ip_pools`

### 14.2.1 命令

```bash
python manage.py sync_geo_ip_pools
```

### 14.2.2 参数

| 参数 | 说明 |
|------|------|
| `--pool china` | 仅同步中国内池 |
| `--pool international` | 仅同步国际池 |
| `--no-fallback` | 禁用多源备份，仅使用主数据源 |

### 14.2.3 行为摘要

1. 读取 `get_settings()` 中的 **`IP_GUARD_GEO_CHINA_POOL_URL`**，拉取文本并构建索引，写入 Redis `ip_guard:geo_pool:data:china`，更新 `IpGeoPoolStatus`（`pool_key=china`）。
2. 若 **`IP_GUARD_GEO_INTERNATIONAL_POOL_URL`** 非空，同样处理 `international` 池；否则 **跳过**（不删除已有 Redis 数据）。
3. **多源备份**：当 `IP_GUARD_GEO_POOL_MULTI_SOURCE_ENABLED=True`（默认）时，自动尝试多个数据源 URL，任一成功即返回，全部失败才报错。
4. 失败时在 `IpGeoPoolStatus.last_error` 记录错误信息。

### 14.2.4 定时任务示例

**crontab**（每日 3:15）：

```cron
15 3 * * * cd /srv/myapp && /srv/venv/bin/python manage.py sync_geo_ip_pools >> /var/log/ip_guard_geo_sync.log 2>&1
```

**systemd timer**（示意）：

```ini
[Unit]
Description=Sync django-ip-safeguard geo IP pools

[Timer]
OnCalendar=*-*-* 03:15:00
Persistent=true

[Install]
WantedBy=timers.target
```

```ini
[Unit]
Description=Run sync_geo_ip_pools

[Service]
Type=oneshot
WorkingDirectory=/srv/myapp
ExecStart=/srv/venv/bin/python manage.py sync_geo_ip_pools
User=www-data
```

### 14.2.5 与控制台的关系

Vue 策略页「**立即同步池**」调用 `POST /ip-guard/api/geo-pools/sync/`，**与命令共享同一套 Python 同步逻辑**，便于紧急更新。

## 14.3 GeoIP2 数据库下载：`download_geoip2_db`

### 14.3.1 命令

```bash
python manage.py download_geoip2_db
```

### 14.3.2 参数

| 参数 | 说明 |
|------|------|
| `--output-dir DIR` | 数据库保存目录（默认: 项目根目录下的 `geoip2_data/`） |
| `--license-key KEY` | MaxMind License Key（从 MaxMind 官网获取，免费注册） |
| `--city-only` | 仅下载 City 数据库 |
| `--asn-only` | 仅下载 ASN 数据库 |
| `--use-mirror` | 优先使用镜像源下载（默认启用） |

### 14.3.3 行为摘要

1. 下载 MaxMind GeoLite2-City.mmdb 和 GeoLite2-ASN.mmdb 到指定目录。
2. **多源下载**：优先尝试国内镜像源（如 gitmirror.com、ghproxy.net），失败后尝试 GitHub Raw 源。
3. 支持 `.mmdb`、`.gz`、`.tar.gz` 格式自动解压。
4. 下载成功后输出配置提示，将路径添加到 `settings.py`。

### 14.3.4 配置示例

```python
# settings.py
IP_GUARD_GEOIP2_CITY_DB_PATH = r'/srv/myapp/geoip2_data/GeoLite2-City.mmdb'
IP_GUARD_GEOIP2_ASN_DB_PATH = r'/srv/myapp/geoip2_data/GeoLite2-ASN.mmdb'
IP_GUARD_GEOIP2_ENABLED = True
```

### 14.3.5 定时更新

GeoLite2 数据库每月更新一次，建议设置定时任务：

```cron
0 4 1 * * cd /srv/myapp && /srv/venv/bin/python manage.py download_geoip2_db --output-dir /srv/myapp/geoip2_data >> /var/log/geoip2_download.log 2>&1
```

## 14.4 威胁情报同步：`sync_threat_intel`

### 14.4.1 命令

```bash
python manage.py sync_threat_intel
```

### 14.4.2 参数

| 参数 | 说明 |
|------|------|
| `--feed NAME` | 仅同步指定情报源（如: `spamhaus_drop`, `tor_exit_nodes`） |
| `--list-feeds` | 列出所有可用的情报源 |
| `--force` | 强制同步，忽略启用状态 |

### 14.4.3 内置情报源

| 情报源名称 | URL | 格式 | 威胁类型 | 自动封禁 |
|-----------|-----|------|---------|---------|
| `spamhaus_drop` | Spamhaus DROP | CIDR列表 | botnet | 是 |
| `spamhaus_edrop` | Spamhaus EDROP | CIDR列表 | botnet | 是 |
| `tor_exit_nodes` | Tor Project | IP列表 | tor | 否 |
| `emerging_threats` | Emerging Threats | IP列表 | malware | 是 |

### 14.4.4 行为摘要

1. 从各情报源拉取最新数据，解析 IP/CIDR 条目。
2. 写入 Redis 缓存（`ip_guard:threat_intel:*`），供本地风险规则引擎使用。
3. 自动封禁标记的 IP 条目直接写入封禁列表。
4. 更新 `ThreatIntelFeedStatus` 数据库表记录同步状态。
5. 支持自定义情报源（通过 `IP_GUARD_THREAT_INTEL_CUSTOM_FEEDS` 配置）。

### 14.4.5 配置示例

```python
# settings.py
IP_GUARD_THREAT_INTEL_ENABLED = True
IP_GUARD_THREAT_INTEL_SPAMHAUS_ENABLED = True
IP_GUARD_THREAT_INTEL_TOR_ENABLED = True
IP_GUARD_THREAT_INTEL_EMERGING_ENABLED = True
IP_GUARD_THREAT_INTEL_CUSTOM_FEEDS = '''
{
    "my_custom_feed": {
        "url": "https://example.com/threats.txt",
        "format": "ip_list",
        "threat_type": "custom",
        "auto_ban": false,
        "ttl": 86400
    }
}
'''
```

### 14.4.6 定时任务

建议每 1-6 小时同步一次：

```cron
0 */4 * * * cd /srv/myapp && /srv/venv/bin/python manage.py sync_threat_intel >> /var/log/threat_intel_sync.log 2>&1
```

## 14.5 IP 信誉快照：`snapshot_ip_reputation`

### 14.5.1 命令

```bash
python manage.py snapshot_ip_reputation
```

### 14.5.2 参数

| 参数 | 说明 |
|------|------|
| `--hours N` | 统计最近 N 小时的数据（默认使用配置中的间隔） |
| `--top-n N` | 仅处理访问次数最高的前 N 个 IP（默认 500） |
| `--dry-run` | 仅显示统计结果，不写入数据库 |

### 14.5.3 行为摘要

1. 从 `IpAccessLog` 中统计活跃 IP 的访问频率。
2. 计算每个 IP 的 1 小时和 24 小时拦截/放行次数。
3. 根据历史快照计算风险趋势（`stable`/`rising`/`declining`）。
4. 写入 `IpReputationHistory` 表，供后台趋势分析使用。
5. `--dry-run` 模式可预览结果而不写入数据库。

### 14.5.4 定时任务

建议每小时执行一次：

```cron
0 * * * * cd /srv/myapp && /srv/venv/bin/python manage.py snapshot_ip_reputation >> /var/log/ip_reputation.log 2>&1
```

## 14.6 完整定时任务配置示例

```cron
# Django IP Safeguard 定时任务
# 每日 3:15 同步地理 IP 池
15 3 * * * cd /srv/myapp && /srv/venv/bin/python manage.py sync_geo_ip_pools >> /var/log/ip_guard_geo_sync.log 2>&1

# 每月 1 日 4:00 更新 GeoIP2 数据库
0 4 1 * * cd /srv/myapp && /srv/venv/bin/python manage.py download_geoip2_db --output-dir /srv/myapp/geoip2_data >> /var/log/geoip2_download.log 2>&1

# 每 4 小时同步威胁情报
0 */4 * * * cd /srv/myapp && /srv/venv/bin/python manage.py sync_threat_intel >> /var/log/threat_intel_sync.log 2>&1

# 每小时生成 IP 信誉快照
0 * * * * cd /srv/myapp && /srv/venv/bin/python manage.py snapshot_ip_reputation >> /var/log/ip_reputation.log 2>&1
```

## 14.8 发布检查清单（与定时任务相关）

- [ ] 生产 `settings` 已配置 Redis 与 Provider
- [ ] 已配置 crontab/timer 执行 `sync_geo_ip_pools`（若启用地理池规则）
- [ ] 已配置 `sync_threat_intel` 定时任务（若启用威胁情报）
- [ ] 已配置 `snapshot_ip_reputation` 定时任务（若启用信誉快照）
- [ ] 已下载 GeoIP2 数据库并配置路径（若启用本地查询）
- [ ] 首次上线已手动执行一次同步并验收 `geo-pools/status`
- [ ] 日志目录权限与轮转（logrotate）
