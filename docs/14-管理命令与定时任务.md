# 14 管理命令与定时任务

## 14.1 Django 迁移

安装或升级插件版本后：

```bash
python manage.py migrate django_ip_safeguard
```

会创建/更新包括但不限于：`IpGuardPolicy`、`IpAccessLog`、`IpBanRecord`、`IpGeoPoolStatus` 等表。

---

## 14.2 地理 IP 池同步：`sync_geo_ip_pools`

### 14.2.1 命令

```bash
python manage.py sync_geo_ip_pools
```

### 14.2.2 行为摘要

1. 读取 `get_settings()` 中的 **`IP_GUARD_GEO_CHINA_POOL_URL`**，拉取文本并构建索引，写入 Redis `ip_guard:geo_pool:data:china`，更新 `IpGeoPoolStatus`（`pool_key=china`）。  
2. 若 **`IP_GUARD_GEO_INTERNATIONAL_POOL_URL`** 非空，同样处理 `international` 池；否则 **跳过**（不删除已有 Redis 数据）。  
3. 失败时在 `IpGeoPoolStatus.last_error` 记录错误信息。

### 14.2.3 定时任务示例

**crontab**（每日 3:15）：

```cron
15 3 * * * cd /srv/myapp && /srv/venv/bin/python manage.py sync_geo_ip_pools >> /var/log/ip_guard_geo_sync.log 2>&1
```

**systemd timer**（示意）：

```ini
[Unit]
Description=Sync django-ip-safeguard geo IP pools

[Timer]
OnCalendar=*-*-* 03:15:00
Persistent=true

[Install]
WantedBy=timers.target
```

```ini
[Unit]
Description=Run sync_geo_ip_pools

[Service]
Type=oneshot
WorkingDirectory=/srv/myapp
ExecStart=/srv/venv/bin/python manage.py sync_geo_ip_pools
User=www-data
```

### 14.2.4 与控制台的关系

Vue 策略页「**立即同步池**」调用 `POST /ip-guard/api/geo-pools/sync/`，**与命令共享同一套 Python 同步逻辑**，便于紧急更新。

## 14.3 其它运维命令（Django 内置）

| 命令 | 用途 |
|------|------|

## 14.4 发布检查清单（与定时任务相关）

- [ ] 生产 `settings` 已配置 Redis 与 Provider  
- [ ] 已配置 crontab/timer 执行 `sync_geo_ip_pools`（若启用地理池规则）  
- [ ] 首次上线已手动执行一次同步并验收 `geo-pools/status`  
- [ ] 日志目录权限与轮转（logrotate）
