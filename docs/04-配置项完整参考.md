# 04 配置项完整参考

所有配置项均在 **Django `settings`** 中以 `IP_GUARD_*` 为主（JWT 另见 `IP_GUARD_JWT_*`）。  
代码读取入口：`django_ip_safeguard.conf.get_settings()`，返回冻结数据类 **`IpGuardSettings`**。

若 **`IP_GUARD_ENABLE_POLICY_CENTER=True`**，则数据库 **`IpGuardPolicy`**（`name=default`）中的部分字段会**覆盖**下表对应项（地理池 **数据源 URL** 仅来自 settings，见 [05](./05-策略中心与数据库模型.md)、[06](./06-地理IP池与定时同步.md)）。

---

## 4.1 总开关与 Redis

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_ENABLED` | bool | `True` | 为 `False` 时中间件完全旁路。 |
| `IP_GUARD_REDIS_URL` | str | `redis://127.0.0.1:6379/0` | 情报、封禁、限流、地理池、熔断计数等共用该连接。 |

---

## 4.2 情报缓存与封禁 TTL

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_CACHE_TTL` | int | `3600` | 名称保留；高风险/低风险 TTL 另见下行。 |
| `IP_GUARD_BAN_TTL` | int | `86400` | 封禁写入 Redis 的默认 TTL（秒）。 |
| `IP_GUARD_HIGH_RISK_CACHE_TTL` | int | `7200` | 风险分 ≥ 阈值时情报缓存秒数。 |
| `IP_GUARD_LOW_RISK_CACHE_TTL` | int | `1800` | 低于阈值时情报缓存秒数。 |

---

## 4.3 Provider（IP 情报）

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_PROVIDER` | str | `"dummy"` | `dummy` 或 `http`。 |
| `IP_GUARD_PROVIDER_ENDPOINT` | str | `""` | `http` 模式下的查询 URL。 |
| `IP_GUARD_PROVIDER_API_KEY` | str | `""` | 可从环境变量 `IP_GUARD_PROVIDER_API_KEY` 注入。 |
| `IP_GUARD_PROVIDER_TIMEOUT` | float | `3.0` | 单次 HTTP 超时（秒）。 |
| `IP_GUARD_PROVIDER_MAX_RETRIES` | int | `2` | 最大重试次数。 |
| `IP_GUARD_PROVIDER_RETRY_BACKOFF` | float | `0.2` | 退避基数（秒）。 |
| `IP_GUARD_PROVIDER_HEADERS` | dict | `{}` | 附加请求头。 |
| `IP_GUARD_PROVIDER_CIRCUIT_BREAKER_FAILURES` | int | `5` | 连续失败达到该次数视为熔断打开。 |
| `IP_GUARD_PROVIDER_CIRCUIT_BREAKER_TTL` | int | `60` | 熔断计数 Redis 键的过期秒数。 |

---

## 4.4 风险引擎相关

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_RISK_SCORE_THRESHOLD` | int | `70` | 大于等于则拦截（并可能触发封禁，视决策）。 |
| `IP_GUARD_BLOCKED_RISK_TAGS` | tuple | `("tor","proxy","vpn")` | 与情报返回标签集合求交，命中则拦截；比较时大小写不敏感。 |
| `IP_GUARD_ALLOWED_COUNTRIES` | tuple | `()` | 非空时：国家码**不在**集合内则拦截（ISO2，如 `CN`）。 |
| `IP_GUARD_BLOCKED_COUNTRIES` | tuple | `()` | 非空时：国家在集合内则拦截。 |

---

## 4.5 IP 列表与限流

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_IP_WHITELIST` | tuple | `()` | 单 IP 或 CIDR；命中直接放行。 |
| `IP_GUARD_IP_BLACKLIST` | tuple | `()` | 单 IP 或 CIDR；命中拦截。 |
| `IP_GUARD_RATE_LIMIT_PER_MINUTE` | int | `0` | `0` 关闭；否则单 IP 60 秒窗口内最大请求数。 |

---

## 4.6 失败放行与代理

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_FAIL_OPEN` | bool | `True` | Provider 失败时默认是否放行。 |
| `IP_GUARD_FAIL_OPEN_PATH_PREFIXES` | tuple | `()` | 路径前缀命中则**强制放行**（在失败场景下）。 |
| `IP_GUARD_FAIL_CLOSE_PATH_PREFIXES` | tuple | `()` | 路径前缀命中则**强制拦截**（在失败场景下）。 |
| `IP_GUARD_TRUSTED_PROXY_CIDRS` | tuple | `()` | 仅当直连 IP 落在这些网段内时，才解析 `X-Forwarded-For` 等头取客户端 IP。 |

路径前缀规则优先级见 [07](./07-IP情报Provider与风险引擎.md) 与中间件实现。

## 4.7 响应与审计脱敏

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_BLOCK_STATUS_CODE` | int | `403` | 拦截时 HTTP 状态码（一般保持 403）。 |
| `IP_GUARD_USE_DB_LOG` | bool | `False` | 是否写入 `IpAccessLog`。 |
| `IP_GUARD_IP_MASK_ENABLED` | bool | `False` | API 返回审计数据时是否脱敏 IP；数据库始终存储完整 IP。 |
| `IP_GUARD_IP_MASK_KEEP_PREFIX` | int | `2` | 脱敏保留前几段（IPv4 点分）；仅当 `IP_GUARD_IP_MASK_ENABLED=True` 时生效。 |

## 4.8 策略中心

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_ENABLE_POLICY_CENTER` | bool | `True` | 关闭时完全以 settings 为准。 |
| `IP_GUARD_POLICY_CACHE_SECONDS` | int | `30` | 进程内策略合并结果缓存秒数。 |

## 4.9 并发与去重

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_DEDUPE_LOCK_SECONDS` | int | `3` | 同一 IP 并发未命中缓存时，仅一个请求去打 Provider，其余短暂失败降级。 |

## 4.10 管理路由前缀

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_ADMIN_URL_PREFIX` | str | `"ip-guard"` | 用于文档与部分路径说明；实际挂载仍以项目 `urls.py` 中 `path()` 为准。 |

## 4.11 JWT（管理 API）

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_JWT_SECRET_KEY` | str | `""`（环境变量 `IP_GUARD_JWT_SECRET_KEY` 亦可） | 使用 JWT 登录/刷新/校验前须配置且 **长度 ≥ 32**；未配置时应用可正常启动，仅 JWT 相关接口会失败。 |
| `IP_GUARD_JWT_ALGORITHM` | str | `"HS256"` | 当前实现固定 HS256。 |
| `IP_GUARD_JWT_ACCESS_TTL` | int | `7200` | access token 有效期（秒）。 |
| `IP_GUARD_JWT_REFRESH_TTL` | int | `604800` | refresh token 有效期（秒）。 |

详见 [10-认证CSRF-JWT与权限模型](./10-认证CSRF-JWT与权限模型.md)。

## 4.12 地理 IP 池

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_GEO_CHINA_POOL_URL` | str | `https://ispip.clang.cn/all_cn_cidr.txt` | 中国内 CIDR 文本源（默认使用国内可达源）。 |
| `IP_GUARD_GEO_INTERNATIONAL_POOL_URL` | str | `""` | 国际（或任意）CIDR 文本源；**空则不同步国际池**。 |
| `IP_GUARD_CHINA_POOL_RULE` | str | `off` | 策略中心关闭时默认中国池规则。 |
| `IP_GUARD_INTERNATIONAL_POOL_RULE` | str | `off` | 策略中心关闭时默认国际池规则。 |
| `IP_GUARD_GEO_POOL_INDEX_CACHE_SECONDS` | int | `60` | Worker 进程内从 Redis 刷新池索引的缓存间隔。 |
| `IP_GUARD_GEO_POOL_MULTI_SOURCE_ENABLED` | bool | `True` | 启用多数据源备份，主源失败自动切换备用源。 |
| `IP_GUARD_GEO_CHINA_POOL_BACKUP_URLS` | tuple | `("https://ispip.clang.cn/all_cn.txt", "https://raw.githubusercontent.com/17mon/china_ip_list/master/china_ip_list.txt")` | 中国池备用数据源 URL 列表。 |
| `IP_GUARD_GEO_INTERNATIONAL_POOL_BACKUP_URLS` | tuple | `()` | 国际池备用数据源 URL 列表。 |

规则取值：`off` | `allow_only_in_pool` | `block_in_pool`。详见 [06](./06-地理IP池与定时同步.md)。

## 4.13 GeoIP2 本地数据库

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_GEOIP2_ENABLED` | bool | `False` | 启用 MaxMind GeoLite2 本地离线查询。 |
| `IP_GUARD_GEOIP2_CITY_DB_PATH` | str | `""` | GeoLite2-City.mmdb 文件路径。 |
| `IP_GUARD_GEOIP2_ASN_DB_PATH` | str | `""` | GeoLite2-ASN.mmdb 文件路径。 |

启用后可零网络延迟获取国家码、城市、经纬度、ASN 等信息。使用 `python manage.py download_geoip2_db` 下载数据库文件。

## 4.14 多 Provider 降级链

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_PROVIDER_CHAIN_ENABLED` | bool | `False` | 启用多 Provider 降级链模式。 |
| `IP_GUARD_PROVIDER_CHAIN_NAMES` | tuple | `()` | Provider 名称列表，按优先级排列。如 `("geoip2", "http", "dummy")`。 |

启用后依次尝试每个 Provider，合并多个来源的情报数据，前者缺失的字段由后者补充。

---

## 4.15 分层缓存

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_L1_CACHE_ENABLED` | bool | `True` | 启用 L1 进程内缓存（极短 TTL，热点 IP 加速）。 |
| `IP_GUARD_L1_CACHE_TTL` | float | `10.0` | L1 缓存 TTL（秒）。 |
| `IP_GUARD_L1_CACHE_MAX_SIZE` | int | `10000` | L1 缓存最大条目数。 |

分层缓存架构：L1（进程内）→ L2（Redis）→ L3（GeoIP2 本地数据库）→ L4（HTTP API Provider）。

---

## 4.16 本地风险规则引擎

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_LOCAL_RISK_ENGINE_ENABLED` | bool | `True` | 启用本地风险规则引擎。 |
| `IP_GUARD_LOCAL_RISK_SUBNET_ATTACK_THRESHOLD` | int | `10` | 同 C 段 IP 数量达到该阈值时触发子网攻击关联。 |

本地规则引擎不依赖外部 API，可检测：Tor 出口节点、僵尸网络 CIDR、数据中心 ASN、代理/VPN 节点、同 C 段攻击关联。

## 4.17 威胁情报订阅

| 配置项 | 类型 | 默认 | 说明 |
|--------|------|------|------|
| `IP_GUARD_THREAT_INTEL_ENABLED` | bool | `False` | 启用威胁情报订阅服务。 |
| `IP_GUARD_THREAT_INTEL_SPAMHAUS_ENABLED` | bool | `True` | 启用 Spamhaus DROP/EDROP 情报源。 |
| `IP_GUARD_THREAT_INTEL_TOR_ENABLED` | bool | `True` | 启用 Tor 出口节点情报源。 |
| `IP_GUARD_THREAT_INTEL_EMERGING_ENABLED` | bool | `True` | 启用 Emerging Threats 情报源。 |
| `IP_GUARD_THREAT_INTEL_CUSTOM_FEEDS` | str | `""` | 自定义情报源 JSON 配置。 |

自定义情报源格式示例：
```json
{
    "my_feed": {
        "url": "https://example.com/threats.txt",
        "format": "ip_list",
        "threat_type": "custom",
        "auto_ban": false,
        "ttl": 86400
    }
}
```
