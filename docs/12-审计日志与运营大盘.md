# 12 审计日志与运营大盘

## 12.1 何时产生审计数据

当 **`IP_GUARD_USE_DB_LOG=True`**（或策略中心库表中 `use_db_log=True`）时，中间件在以下场景调用 `log_access_decision` 写入 **`IpAccessLog`**：

- IP 黑名单拦截  
- 地理池规则拦截  
- 单 IP 限流拦截  
- 封禁命中（若也写日志，视实现）  
- 风险引擎拦截 / 放行  

字段主要包括：IP（数据库始终存储完整 IP，API 返回时按 `IP_GUARD_IP_MASK_ENABLED` 配置脱敏）、关联 **`user_id` / `username`**（未登录为空）、**HTTP `method`**、国家码、**`country_name` / `region` / `city`**（视情报源而定）、风险分、风险标签、决策（`allow`/`block`）、原因、请求路径、时间戳等。

**性能提示**：高 QPS 站点开启写库会带来显著数据库压力，建议配合采样、异步队列或外部日志系统评估。

---

## 12.2 模型：`IpAccessLog`

用途：合规审计、运营分析、事后溯源。  
列表与导出接口见 [09-管理REST-API参考](./09-管理REST-API参考.md) 中「审计日志」章节。

筛选能力（`GET /ip-guard/api/access-logs/`）：

- `decision`：`allow` / `block`  
- `country`：ISO2 大写  
- `path`：路径子串  
- `q`：IP 子串  
- `username`：用户名子串（模糊）  
- `user_id`：Django 用户主键（精确，数字）  
- `start` / `end`：`YYYY-MM-DD` 日期范围  

**按用户汇总**（需开启写库）：`GET /ip-guard/api/access-logs/user-summary/?user_id=<id>&days=<1-180>`，返回该用户在窗口内的访问总次数、**全量不同 IP 数**，以及按 IP 聚合的请求次数与最近一条日志上的地理信息（表格最多展示前 200 个 IP，按次数降序）。`decision=security_audit` 的记录不计入统计。

---

## 12.3 CSV 导出

- 路径：`GET /ip-guard/api/access-logs/export/`  
- 与列表接口 **相同筛选参数**  
- **最多 10000 条**，UTF-8 带 BOM，便于 Excel 打开。  
- JWT 需在原始 HTTP 客户端上携带 `Authorization`。

---

## 12.4 运营大盘：`/ip-guard/api/dashboard/`

提供约 **24 小时** 窗口的聚合指标（具体字段以实现为准），通常包括：

- 总访问量、拦截量、放行量、拦截率  
- 按小时趋势  
- 国家/地区分布（`country_code`）  
- 热门路径 Top  
- 拦截原因 Top  
- 高风险 IP Top  

### 12.4.1 Dashboard 页面布局

Dashboard 页面从上到下分为以下区域：

**① 统计卡片行（4 列）**

| 卡片 | 含义 | 点击跳转 |
|------|------|----------|
| 24h 访问总次数 | 24 小时内所有请求计数 | 审计日志（全部） |
| 24h 放行次数 | decision=allow 的请求数 | 审计日志（decision=allow） |
| 24h 被拦截次数 | decision=block 的请求数 | 审计日志（decision=block） |
| 24h 拦截率 | block / total × 100% | 审计日志（decision=block） |

每个卡片左侧有彩色标识条（蓝/绿/红/黄），hover 时上浮阴影效果，点击跳转到审计日志页并自动填充筛选条件。

**② 全球 IP 访问与攻击分布图**

- 左侧：ECharts 世界地图，按国家请求量着色（浅蓝→深蓝渐变），支持缩放、拖拽
  - **攻击标点**：有攻击记录的国家显示红色涟漪圆点（effectScatter），攻击量越大圆点越大
  - 鼠标悬停显示国家中文名、总请求量、攻击量（红色）、允许量（绿色）
  - 地图标题：「全球 IP 访问与攻击分布」，副标题说明热力=总请求、红点=攻击来源
- 右侧：Top 15 国家/地区堆叠柱状图（绿色=允许、红色=攻击）
  - 鼠标悬停显示允许量和攻击量的详细数值
- **点击国家/标点** → 跳转审计日志页，自动按国家码筛选
- **点击柱状图** → 同样跳转审计日志

国家码→中文名映射覆盖 50+ 常见国家（CN→中国、US→美国、RU→俄罗斯等），未映射的显示原始 ISO2 码。

**国家码数据来源**：
1. IP 情报 Provider（如 GeoIP2 本地库、HTTP Provider）直接返回 `country_code`
2. 当 Provider 返回空或 `UNKNOWN` 时，中间件自动利用已同步的地理 IP 池（如中国池 CIDR）推断国家码
3. 本地回环地址（127.0.0.1）标记为 `LOCAL`，不在地图上显示

**Dashboard API `country_distribution` 字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| country_code | string | ISO2 国家码（如 CN、US） |
| country_name | string | 国家中文名（如 中国、美国） |
| count | int | 总请求数 |
| blocked | int | 被拦截（攻击）次数 |
| allowed | int | 被放行次数 |

API 自动过滤 `UNKNOWN`、`LOCAL`、空值国家码，最多返回 Top 30 国家。

**③ 高风险 IP / 国家分布表**

- 高风险 IP 表：点击行 → 跳转审计日志（decision=block，IP 筛选）
- 国家分布表：显示国家码、国名、总量、攻击量（红色高亮）、允许量（绿色），点击行 → 跳转审计日志（国家码筛选）

**④ 热门路径 / 拦截原因表**

- 热门路径表：点击行 → 跳转审计日志（路径筛选）
- 拦截原因表：仅展示数据

**⑤ 近期记录**

详见 12.5 节。统计卡片（访问总次数/拦截次数/放行次数/封禁事件数）可点击跳转审计日志；攻击记录、访问记录、封禁事件表格行均可点击跳转。

**⑥ 决策分布 / 小时趋势**

纯数据展示表格。

### 12.4.2 跳转到审计日志的参数映射

Dashboard 中所有可点击元素跳转到 `/logs` 页面时，通过 URL query 参数自动填充筛选条件：

| Dashboard 点击 | 跳转 URL 参数 |
|----------------|---------------|
| 统计卡片「放行」 | `?decision=allow` |
| 统计卡片「拦截」 | `?decision=block` |
| 地图/柱状图点击国家 | `?country=CN` |
| 高风险 IP 行 | `?decision=block&q=1.2.3.4` |
| 国家分布行 | `?country=US` |
| 热门路径行 | `?path=/api/login` |

前端 **世界地图** 使用 `country_distribution` 与 `@surbowl/world-geo-json-zh` 对齐 `iso_a2`。

## 12.5 近期记录：`/ip-guard/api/recent-records/`

面向「安全运营日报」场景，支持：

- 可配置天数 `days`（默认 7，最大 30）  
- 按日汇总 allow/block  
- 最近拦截样本、最近访问样本、近期封禁样本（各自 limit 可配）

权限：`view_ipaccesslog` **或** `view_ipbanrecord`（满足其一）。

---

## 12.6 模型：`IpBanRecord`

记录人工封禁、策略触发的封禁等（字段含 `ip`、`ban_reason`、`ban_source`、`is_active`、`expired_at` 等）。  
与 Redis `ip_guard:ban:{ip}` 配合：解封 API 会同步更新库表 `is_active=False`。
