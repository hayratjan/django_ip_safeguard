# 12 审计日志与运营大盘

## 12.1 何时产生审计数据

当 **`IP_GUARD_USE_DB_LOG=True`**（或策略中心库表中 `use_db_log=True`）时，中间件在以下场景调用 `log_access_decision` 写入 **`IpAccessLog`**：

- IP 黑名单拦截  
- 地理池规则拦截  
- 单 IP 限流拦截  
- 封禁命中（若也写日志，视实现）  
- 风险引擎拦截 / 放行  

字段主要包括：IP（可脱敏）、国家码、风险分、风险标签、决策（`allow`/`block`）、原因、请求路径、时间戳等。

**性能提示**：高 QPS 站点开启写库会带来显著数据库压力，建议配合采样、异步队列或外部日志系统评估。

---

## 12.2 模型：`IpAccessLog`

用途：合规审计、运营分析、事后溯源。  
列表与导出接口见 [09-管理REST-API参考](./09-管理REST-API参考.md) 中「审计日志」章节。

筛选能力（`GET /ip-guard/api/access-logs/`）：

- `decision`：`allow` / `block`  
- `country`：ISO2 大写  
- `path`：路径子串  
- `q`：IP 子串  
- `start` / `end`：`YYYY-MM-DD` 日期范围  

---

## 12.3 CSV 导出

- 路径：`GET /ip-guard/api/access-logs/export/`  
- 与列表接口 **相同筛选参数**  
- **最多 10000 条**，UTF-8 带 BOM，便于 Excel 打开。  
- JWT 需在原始 HTTP 客户端上携带 `Authorization`。

---

## 12.4 运营大盘：`/ip-guard/api/dashboard/`

提供约 **24 小时** 窗口的聚合指标（具体字段以实现为准），通常包括：

- 总访问量、拦截量、拦截率  
- 按小时趋势  
- 国家/地区分布（`country_code`）  
- 热门路径 Top  
- 拦截原因 Top  

前端 **世界地图** 使用 `country_distribution` 与 `@surbowl/world-geo-json-zh` 对齐 `iso_a2`。

---

## 12.5 近期记录：`/ip-guard/api/recent-records/`

面向「安全运营日报」场景，支持：

- 可配置天数 `days`（默认 7，最大 30）  
- 按日汇总 allow/block  
- 最近拦截样本、最近访问样本、近期封禁样本（各自 limit 可配）

权限：`view_ipaccesslog` **或** `view_ipbanrecord`（满足其一）。

---

## 12.6 模型：`IpBanRecord`

记录人工封禁、策略触发的封禁等（字段含 `ip`、`ban_reason`、`ban_source`、`is_active`、`expired_at` 等）。  
与 Redis `ip_guard:ban:{ip}` 配合：解封 API 会同步更新库表 `is_active=False`。
