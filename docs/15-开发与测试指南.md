# 15 开发与测试指南

## 15.1 开发环境准备

```bash
git clone <repository>
cd django_ip_safegurad
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

依赖版本要求见 `pyproject.toml`（Python ≥3.9，Django ≥4.2）。

---

## 15.2 运行测试

```bash
python3 -m pytest tests/ -q
```

测试目录：`tests/`，包含配置、中间件工具函数、地理池索引、缓存、策略等单元测试。  
若需 Django DB 测试，可在后续引入 `pytest-django` 与 `django_db` 标记扩展（当前以纯单元与轻量逻辑为主）。

---

## 15.3 静态检查

```bash
python3 -m ruff check django_ip_safeguard tests
```

行宽与目标 Python 版本见 `pyproject.toml` 中 `[tool.ruff]`。

---

## 15.4 代码结构导读（扩展阅读）

| 路径 | 职责 |
|------|------|
| `middleware.py` | 请求编排 |
| `conf.py` | 配置模型与读取 |
| `services/policy_service.py` | 策略中心合并与缓存 |
| `services/cache.py` | Redis 封装 |
| `services/risk_engine.py` | 风险规则 |
| `services/geo_ip_pool_*.py` | 地理池索引、运行时、同步 |
| `views.py` / `urls.py` | 管理 API 与 Dashboard 页 |
| `frontend-admin/` | Vue 控制台 |

---

## 15.5 扩展 Provider

1. 实现 `BaseIpIntelProvider` 子类，返回标准 `IpIntel`。  
2. 在 `provider_factory.build_provider` 中注册新类型字符串。  
3. 为 `IP_GUARD_PROVIDER` 增加文档与单测 mock。  

HTTP 以外的协议（gRPC 等）可在同一抽象下扩展，注意超时与熔断行为与中间件兼容。
