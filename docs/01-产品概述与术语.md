# 01 产品概述与术语

## 1.1 产品定位

**django-ip-safeguard**（PyPI 包名）是一个面向 Django 4.2+ 的**请求前置 IP 安全插件**。它在请求进入业务视图之前，基于客户端 IP 完成：

- 可选的 **IP 白名单 / 黑名单**（支持 CIDR）；
- 可选的 **中国内 / 国际 CIDR 地理池**规则（数据由远程列表同步至 Redis）；
- **单 IP 每分钟请求上限**（Redis 计数）；
- **封禁状态**（Redis，可带 TTL）；
- **第三方或本地的 IP 情报**（风险分、风险标签、国家码等）；
- **风险引擎**综合判定放行或拦截；
- 可选的 **数据库审计**（`IpAccessLog`）与 **运营 API / Vue 控制台**。

插件**不替代** WAF、DDoS 清洗、WAF Bot 管理等业务线安全产品；它解决的是「在 Django 进程内、以配置驱动的方式，对 HTTP 请求做 IP 维度的一致策略与可观测性」。

## 1.2 核心术语

| 术语 | 含义 |
|------|------|
| **客户端 IP** | 经 `X-Forwarded-For` 等头解析后的「认为发起请求」的 IP；受 `IP_GUARD_TRUSTED_PROXY_CIDRS` 约束，仅在受信代理条件下才信任转发头。 |
| **运行时策略** | `get_settings()` 与数据库 `IpGuardPolicy`（若启用策略中心）合并后的不可变配置对象 `IpGuardSettings`。 |
| **策略中心** | `IP_GUARD_ENABLE_POLICY_CENTER=True` 时，以库表 `IpGuardPolicy`（默认 `name=default`）覆盖部分 `settings` 项；合并结果带进程内短缓存。 |
| **Provider** | IP 情报提供者：`dummy`（本地占位）或 `http`（HTTP 接口）。 |
| **风险引擎** | `evaluate_ip_risk`：在已有 `IpIntel` 上应用风险分阈值、风险标签黑名单、国家白/黑名单。 |
| **失败放行（fail-open）** | Provider 不可用或异常时，是否仍允许请求进入后续 Django 处理链；可按全局与路径前缀细分。 |
| **地理 IP 池** | 从 URL 拉取的纯文本 CIDR 列表，经合并后存 Redis；策略上分为 **中国内池（china）** 与 **国际池（international）** 两套索引。 |
| **Staff / 权限点** | 管理 API 要求 `is_staff`，并校验 Django 模型权限（如 `django_ip_safeguard.change_ipguardpolicy`）；JWT 与 Session 用户均需满足。 |

---

## 1.3 与 Django 请求生命周期的关系

1. Django 中间件链执行到 **`IpGuardMiddleware`**。  
2. 若 `IP_GUARD_ENABLED=False` 或合并后 `enabled=False`，中间件直接放行。  
3. 否则按 [03-中间件与请求判定流程](./03-中间件与请求判定流程.md) 的顺序判定。  
4. 仅当所有本地策略均放行时，才调用 `get_response`，进入后续中间件与视图。

**注意**：被拦截的请求返回 JSON 响应（状态码默认 403，可配置），**不会**进入业务视图；因此业务层无需再写重复的 IP 黑名单逻辑（除非有特殊灰度需求）。

## 1.4 企业版能力边界（功能清单）

| 能力 | 说明 |
|------|------|
| 中间件编排 | 白名单 → 黑名单 → 地理池 → 限流 → 封禁 → 情报 → 风险引擎 |
| 策略中心 | 数据库策略覆盖 settings，支持热更新（受缓存秒数限制） |
| 地理池 | 中国默认源可配；国际源需自建 URL；管理命令 + API 手动同步 |
| 运营 API | Dashboard、近期记录、策略、封禁、审计、健康、地理池状态等 |
| 前端控制台 | Vue3 + Element Plus，Session/JWT 双模式 |
| 认证 | Session + CSRF；JWT access/refresh；权限驱动菜单与路由 |
| 审计 | 可选写库 + CSV 导出（上限与筛选见 API 文档） |
