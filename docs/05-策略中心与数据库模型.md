# 05 策略中心与数据库模型

## 5.1 策略中心是什么

当 **`IP_GUARD_ENABLE_POLICY_CENTER=True`**（默认）时，插件在每次请求中会通过 **`load_effective_policy(base_config)`** 将：

- **基础配置**：来自 `get_settings()`（即 Django `settings` 中的 `IP_GUARD_*` 等）；  
- **数据库策略**：表 **`IpGuardPolicy`** 中 `name="default"` 的一条记录；

合并为**运行时策略**（仍为 `IpGuardSettings` 结构，不可变）。

若库中不存在 `default` 策略，则运行时等于基础配置。

---

## 5.2 合并字段一览（库表覆盖 settings）

以下字段在启用策略中心时，**以数据库 `IpGuardPolicy` 为准**（与 `policy_service.load_effective_policy` 实现一致）：

| 字段 | 说明 |
|------|------|
| `enabled` | 是否启用防护 |
| `risk_score_threshold` | 风险分阈值 |
| `blocked_risk_tags` | 风险标签黑名单（存 JSON 列表，合并时转大写元组） |
| `allowed_countries` / `blocked_countries` | 国家白/黑名单（ISO2 大写） |
| `country_mode` | 国家策略模式：`default`（兼容旧版白名单优先）、`allowlist`（仅允许列表内国家）、`blacklist`（仅拦截列表内国家） |
| `block_unknown_country` | 在 `allowlist` 模式下，是否拦截无法解析国家的请求 |
| `ip_whitelist` / `ip_blacklist` | IP 白/黑名单（字符串列表，支持 CIDR） |
| `rate_limit_per_minute` | 单 IP 每分钟上限 |
| `fail_open` | 全局失败放行 |
| `fail_open_path_prefixes` / `fail_close_path_prefixes` | 路径级失败放行/阻断 |
| `block_status_code` | 拦截 HTTP 状态码 |
| `cache_ttl` / `ban_ttl` | 情报默认 TTL 与封禁 TTL（与中间件/写库逻辑相关项） |
| `use_db_log` | 是否写 `IpAccessLog` |
| `china_pool_rule` / `international_pool_rule` | 地理池规则 |

**仍仅来自 settings、不由库表覆盖的项**包括但不限于：

- `redis_url`、`provider` 系列、`trusted_proxy_cidrs`  
- **地理池数据源 URL**：`geo_china_pool_url`、`geo_international_pool_url`  
- `geo_pool_index_cache_seconds`  
- JWT 相关密钥与 TTL  

这样设计的原因：**数据源 URL 往往随环境/密钥管理变化**，且列表体积大，适合留在配置中心或 `settings`；库表只管**业务策略开关与阈值**。

---

## 5.3 进程内策略缓存

- 变量：`policy_service._POLICY_CACHE`  
- 行为：在 `policy_cache_seconds` 内复用同一份合并结果，减少每次请求查库。  
- **失效**：策略 API POST 保存后会调用 **`invalidate_policy_cache()`**；Django Admin 中保存 `IpGuardPolicy` 时也会钩子失效。

若你在 **shell / 任务 / 多进程** 中直接 `IpGuardPolicy.objects.update(...)`，需自行调用 `invalidate_policy_cache()` 或等待缓存自然过期。

## 5.4 模型：`IpGuardPolicy`

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | str，唯一 | 固定使用 `default` 作为企业默认策略名 |
| `enabled` | bool | 总开关 |
| `risk_score_threshold` | int | 风险分阈值 |
| `blocked_risk_tags` | JSON 数组 | 如 `["TOR","VPN"]` |
| `allowed_countries` / `blocked_countries` | JSON 数组 | 两位国家码 |
| `country_mode` | str | `default` / `allowlist` / `blacklist` |
| `block_unknown_country` | bool | 允许列表模式下是否拦截未知国家 |
| `ip_whitelist` / `ip_blacklist` | JSON 数组 | 单 IP 或 CIDR 字符串 |
| `rate_limit_per_minute` | int | `0` 关闭 |
| `fail_open` | bool | Provider 失败默认是否放行 |
| `fail_open_path_prefixes` / `fail_close_path_prefixes` | JSON 数组 | 路径字符串 |
| `block_status_code` | int | 建议 403 |
| `cache_ttl` / `ban_ttl` | int | 秒 |
| `use_db_log` | bool | 审计写库 |
| `china_pool_rule` / `international_pool_rule` | str | 见 [06](./06-地理IP池与定时同步.md) |
| `updated_at` | datetime | 自动更新 |

**管理接口**：`GET/POST /ip-guard/api/policy/`（权限见 [09](./09-管理REST-API参考.md)）。POST 时字段校验：国家码格式、IP/CIDR 合法、地理池规则枚举等。

---

## 5.5 模型：`IpGeoPoolStatus`

记录地理池**同步元数据**（行数、区间数、上次成功时间、错误信息）。**大体积 CIDR 索引在 Redis**，不在此表。

详见 [06-地理IP池与定时同步](./06-地理IP池与定时同步.md)。

---

## 5.6 模型：`IpAccessLog` / `IpBanRecord`

- **`IpAccessLog`**：可选审计，字段含 IP（可脱敏）、`user`/`username`、`method`、国家码与国名/地区/城市、风险分、标签、决策、原因、路径、时间等。  
- **`IpBanRecord`**：封禁事件记录（与 Redis 封禁键配合，供运营列表展示）。

详见 [12-审计日志与运营大盘](./12-审计日志与运营大盘.md)。
